# Specs

Every change in this app should start as a spec or feature document.

## Rule

Do not move to the next feature until the current spec is:

- implemented
- visually validated
- built successfully
- checked for regressions
- reviewed for unnecessary complexity or performance issues

After that:

- commit the spec work with a conventional commit
- push the branch/repo update

## Suggested Naming

- `specs/001-home-hero-tuning.md`
- `specs/002-project-content-replacement.md`
- `specs/003-mobile-nav-improvement.md`

Use increasing numeric prefixes to keep features ordered.
